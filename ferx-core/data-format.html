Data Format

ferx-core reads data in NONMEM-compatible CSV format. This is the standard format shared by NONMEM, Monolix, and most population PK tools.

Required columns

Column Type Description
ID string/numeric Subject identifier
TIME numeric Time relative to first dose
DV numeric Dependent variable (observed concentration)

Optional standard columns

Column Type Default Description
EVID integer 0 Event ID — see table below
AMT numeric 0 Dose amount (dose rows only)
CMT integer 1 Compartment number (1-indexed)
RATE numeric 0 Infusion rate; 0 = bolus
MDV integer 0 Missing DV flag; 1 = skip this observation
II numeric 0 Interdose interval for steady-state records
SS integer 0 Steady-state flag; 1 = assume steady state before TIME=0
CENS integer 0 BLOQ flag; 1 = DV is below LLOQ (DV carries the LLOQ value)

Missing values may be represented as . or as an empty cell.

Column names are case-insensitive: ID, Id, and id are all recognised.

Event types (EVID)

EVID Meaning
0 Observation record. DV is used for estimation.
1 Dose record. AMT is administered to compartment CMT.
2 Covariate-change marker. Compartment amounts are unchanged but the PK parameter expressions are re-evaluated using this row’s covariate values. No-op when all covariates are time-constant.
4 Reset + dose. All compartment amounts are zeroed before dosing.

Covariate columns

Any column not in the standard set above is treated as a covariate.

  • Time-constant covariates: the first non-missing value per subject is used.
  • Time-varying covariates: Last Observation Carried Forward (LOCF) per row — individual-parameter expressions are re-evaluated at each dose and observation row using that row’s covariate snapshot.

Covariate names are matched case-insensitively against names in [individual_parameters] expressions. Numeric covariate columns that are not the IOV occasion column are candidates for ferx_eta_cov() in the R layer.

Time-varying covariate support

Time-varying covariates are supported for all analytical models (1/2/3-cpt) and all ODE-defined models. The Enzyme gradient fast path is also event-driven for TV-covariate subjects on analytical models — the AD path does not fall back to finite differences when covariates vary.

Occasion column (IOV)

When using Inter-Occasion Variability, add an integer occasion-index column and name it with iov_column in [fit_options]:

[fit_options]
  iov_column = OCC

The OCC column:

  • Contains positive integer occasion indices (e.g. 1, 2, 3…) — one per row
  • Applies to both dose and observation rows
  • Is excluded from covariate auto-detection
  • Does not need to start at 1 or be consecutive; different numbers mean different occasions with independent kappa EBEs

BLOQ observations

Set CENS = 1 on any row where the measured concentration is below the lower limit of quantification. The DV column must contain the LLOQ value (not a zero or missing). Pair with bloq_method = m3 in [fit_options] to enable Beal M3 likelihood-based BLOQ handling; the default drop simply discards these rows.

Infusion doses

Set RATE > 0 on a dose row to specify an infusion. Duration is derived as AMT / RATE:

ID,TIME,DV,EVID,AMT,CMT,RATE,MDV
1,0,.,1,500,1,50,1

This administers 500 units over 10 hours at a rate of 50 units/hour.

Dose and infusion routing on analytical models:

CMT IV models Oral models
1 Central compartment Depot — bolus is absorbed via first-order KA
2 Peripheral 1 infusion (2-cpt and 3-cpt) Central compartment infusion
3 Peripheral 2 infusion (3-cpt only) — (panics with a clear message)

Steady-state dosing

Set SS = 1 and II to the dosing interval on a dose row to instruct ferx-core to compute the compartment amounts that would result from an infinite history of doses spaced II apart before TIME = 0:

ID,TIME,DV,EVID,AMT,CMT,SS,II,MDV
1,0,.,1,100,1,1,12,1
1,0.5,25.3,0,.,.,.,.,0

Multiple doses

Each dose is a separate row with EVID = 1. Analytical models sum contributions via multi-dose superposition:

ID,TIME,DV,EVID,AMT,CMT,MDV
1,0,.,1,100,1,1
1,0.5,9.49,0,.,.,0
1,12,.,1,100,1,1
1,12.5,15.2,0,.,.,0
1,24,.,1,100,1,1
1,24.5,18.1,0,.,.,0

Full example with covariates

ID,TIME,DV,EVID,AMT,CMT,RATE,MDV,WT,CRCL
1,0,.,1,100,1,0,1,70,95
1,0.5,9.49,0,.,.,.,0,70,95
1,1,14.42,0,.,.,.,0,70,95
1,2,17.56,0,.,.,.,0,70,95
1,4,15.23,0,.,.,.,0,70,95
1,8,10.15,0,.,.,.,0,70,95
2,0,.,1,150,1,0,1,85,110
2,0.5,14.2,0,.,.,.,0,85,110
2,1,21.3,0,.,.,.,0,85,110
  • Dose records (EVID=1): MDV=1, DV=.
  • Observation records (EVID=0): MDV=0, valid DV
  • Covariates (WT, CRCL) are extra columns; matched to [individual_parameters] by name
