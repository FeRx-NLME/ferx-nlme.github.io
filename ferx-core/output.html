Output Reference

A ferx-core run produces diagnostic data and parameter estimates that are exposed to R through the FitResult struct. The R layer formats these as data frames, named lists, and S3 objects. The CLI writes them as CSV and YAML files.

sdtab — per-observation diagnostics

FitResult::sdtab is a Vec<SdtabRow> exposed to R as a data frame via sdtab_to_dataframe(). The CLI writes it as {model}-sdtab.csv.

Columns

Column Description
ID Subject identifier
TIME Observation time
DV Observed value
PRED Population prediction (ETA = 0)
IPRED Individual prediction (ETA = EBE)
CWRES Conditional weighted residual
IWRES Individual weighted residual
ETA1, ETA2, … Empirical Bayes estimates of each random effect
EBE_OFV Subject’s contribution to the total OFV
N_OBS Number of observations for the subject

Residual definitions

IWRES (Individual Weighted Residual):

\[\text{IWRES}_j = \frac{y_j - \text{IPRED}_j}{\sqrt{V_j}}\]

where \(V_j\) is the residual variance evaluated at the individual prediction (\(\hat{\eta}\)). Measures how well the individual model fits each observation.

CWRES (Conditional Weighted Residual):

\[\text{CWRES}_j = \frac{y_j - f_{0,j}}{\sqrt{\tilde{R}_{jj}}}\]

where \(f_0 = f(\hat{\eta}) - H\hat{\eta}\) is the linearised population prediction and \(\tilde{R} = H\Omega H^T + R\) is the conditional variance matrix. CWRES accounts for the linearisation; it is the residual most appropriate for assessing population model fit.

Shrinkage

Reported in FitResult after every fit. High shrinkage (near 1) indicates that the data are not informative about a random effect — individual ETAs collapse toward zero.

ETA shrinkage (per random effect, shrinkage_eta: Vec<f64>):

\[\text{shrinkage}_k = 1 - \frac{\text{SD}(\hat{\eta}_k)}{\sqrt{\omega_{kk}}}\]

EPS shrinkage (scalar, shrinkage_eps: f64):

\[\text{shrinkage}_\varepsilon = 1 - \text{SD}(\text{IWRES})\]

Values of NaN indicate a zero-variance omega component (ETA) or fewer than two valid residuals (EPS).

Fit YAML ({model}-fit.yaml)

The CLI writes a YAML summary alongside the sdtab. Structure:

model:
  converged: true
  method: focei
objective_function:
  ofv: -280.1838
  aic: -266.1838
  bic: -247.2804
data:
  n_subjects: 10
  n_observations: 110
  n_parameters: 7
theta:
  TVCL:
    estimate: 0.132735
    se: 0.014549
    rse_pct: 11.0
  TVV:
    estimate: 7.694842
    se: 0.293028
    rse_pct: 3.8
omega:
  omega_11:
    variance: 0.028584
    cv_pct: 16.9
    se: 0.006394
  omega_22:
    variance: 0.009613
    cv_pct: 9.8
    se: 0.002165
sigma:
  sigma_1:
    estimate: 0.010638
    se: 0.000788

Key fields:

  • ofv — −2 log-likelihood
  • aic / bic — OFV + 2p / OFV + p·ln(N); p = estimated parameters, N = observations
  • rse_pct — relative SE as percentage: |SE / estimate| × 100
  • cv_pct (omega) — coefficient of variation: \(100\sqrt{\omega_{kk}}\)
  • se — standard error from the finite-difference covariance step

Covariance status

FitResult::covariance_status is one of:

Value Meaning
Computed Covariance step succeeded; SE values are valid
Failed Hessian was singular or inversion failed; SE fields are None
NotRequested covariance = false; SE fields are None

Run record fields

Field Description
model_name Name from the .ferx file (or "Unnamed")
ferx_version Version of ferx-nlme that produced the result
wall_time_secs Wall-clock time for the complete fit (seconds)
gradient_used Gradient method resolved at runtime: "enzyme", "dual", or "finite_differences"
uses_ode_solver true for ODE models, false for analytical PK
n_threads_used Number of rayon threads during estimation

EBE convergence diagnostics

Counters from the inner-loop (EBE) optimizer; always 0 for SAEM.

Field Description
ebe_convergence_warnings Outer iterations in which at least one subject had an unconverged EBE
max_unconverged_subjects Worst-case unconverged count in a single outer iteration
total_ebe_fallbacks Times the Nelder-Mead fallback was invoked across all subjects and iterations

If ebe_convergence_warnings is large relative to maxiter, the inner-loop tolerance (inner_tol) may be too tight, or a subject’s data are poorly compatible with the model.

Optimizer trace CSV

When optimizer_trace = true in [fit_options], ferx-core writes a per-iteration CSV to /tmp/ferx_trace_<pid>_<ts>.csv; the path is stored in FitResult::trace_path.

Each row is one outer iteration. Unused columns contain NA.

Column Populated by Description
iter all Iteration number
method all foce, focei, gn, gn_hybrid, saem
phase gn_hybrid, saem explore / converge / focei (GN polish)
ofv all Objective function value
wall_ms all Wall time for this iteration (ms)
grad_norm BFGS, NLopt gradient-mode Gradient norm
step_norm BFGS Step size
optimizer FOCE/FOCEI Active NLopt algorithm
lm_lambda GN Levenberg-Marquardt damping factor
ofv_delta GN Change in OFV from previous iteration
step_accepted GN Whether the GN step was accepted
cond_nll SAEM Conditional observation NLL
gamma SAEM SAEM step-size (1 in exploration, 1/k in convergence)
mh_accept_rate SAEM Mean MH acceptance rate across subjects
n_ebe_unconverged FOCE/FOCEI Subjects whose inner optimizer did not converge
n_ebe_fallback FOCE/FOCEI Subjects that fell back to Nelder-Mead

The trace is buffered and flushed at fit end; the tail may be lost if the process is killed mid-run.

In R:

fit <- ferx_fit("model.ferx", "data.csv", optimizer_trace = TRUE)
trace <- read.csv(fit$trace_path)
plot(trace$iter, trace$ofv, type = "l", xlab = "Iteration", ylab = "OFV")
