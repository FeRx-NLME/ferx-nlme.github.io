Architecture

Data flow

.ferx model file  ──► parser/model_parser.rs  ──► CompiledModel
NONMEM CSV        ──► io/datareader.rs         ──► Population
                                                        │
                                               api::fit()
                                                        │
                              ┌─────────────────────────┤
                              │                         │
                    outer_optimizer            inner_optimizer
                    (population θ, Ω, σ)     (per-subject EBE η)
                              │                         │
                         pk/  or  ode/           stats/likelihood
                    (predictions)              (NLL evaluation)
                              │
                         FitResult  ──► R via extendr FFI

The outer optimizer iterates over population parameters (theta, omega, sigma). For each candidate parameter set it calls the inner optimizer once per subject to find the empirical Bayes estimate (EBE) of that subject’s random effects. Predictions are routed to the analytical PK models in pk/ or the RK45 ODE solver in ode/, depending on whether the model file has an [odes] block.

FFI boundary

ferx-r wraps ferx-core via extendr. The glue layer in src/rust/src/lib.rs (ferx-r repo) exposes three #[extendr] functions:

  • ferx_rust_fit — calls api::run_model_with_data, returns a named R list
  • ferx_rust_simulate — calls api::simulate, returns an R data frame
  • ferx_rust_predict — calls api::predict, returns an R data frame

Conversion helpers:

  • fit_result_to_list() — unpacks FitResult into an R list (theta, omega, sigma, SEs, OFV/AIC/BIC, diagnostics, model structure, warnings)
  • sdtab_to_dataframe() — converts Vec<SdtabRow> into a data frame with columns PRED, IPRED, CWRES, IWRES, and one column per ETA

Key types

Subject

Holds all per-subject data after the NONMEM CSV is parsed.

pub struct Subject {
    pub id: String,
    pub doses: Vec<DoseEvent>,
    pub obs_times: Vec<f64>,
    pub observations: Vec<f64>,
    pub obs_cmts: Vec<usize>,
    pub covariates: HashMap<String, f64>,           // static (first non-missing)
    pub dose_covariates: Vec<HashMap<String, f64>>, // LOCF snapshot per dose event
    pub obs_covariates: Vec<HashMap<String, f64>>,  // LOCF snapshot per observation
    pub cens: Vec<u8>,       // 0 = observed, 1 = BLOQ (LLOQ stored in observations)
    pub occasions: Vec<u32>, // IOV occasion index per observation; empty if no IOV
    // ...
}

When a subject has no time-varying covariates, dose_covariates and obs_covariates are empty and callers fall through to the static covariates map. Methods obs_cov(j) / dose_cov(k) encapsulate this fallback.

DoseEvent

pub struct DoseEvent {
    pub time: f64,
    pub amt: f64,
    pub cmt: usize,
    pub rate: f64,
    pub duration: f64,  // derived: amt / rate when rate > 0, else 0
    pub ss: bool,
    pub ii: f64,
}

PkParams

Fixed-size array instead of HashMap<String, f64> — required for Enzyme AD.

pub struct PkParams {
    pub values: [f64; 8],  // MAX_PK_PARAMS = 8
}
// Index:  0=CL  1=V  2=Q  3=V2  4=KA  5=F  6=Q3  7=V3

F defaults to 1.0. Unused slots default to 0.0.

ModelParameters

The full set of population parameters under estimation.

pub struct ModelParameters {
    pub theta: Vec<f64>,
    pub theta_names: Vec<String>,
    pub theta_lower: Vec<f64>,
    pub theta_upper: Vec<f64>,
    pub theta_fixed: Vec<bool>,
    pub omega: OmegaMatrix,  // caches Cholesky factor + free_mask
    pub sigma: SigmaVector,
    // ...
}

OmegaMatrix caches the Cholesky factor alongside the full matrix and carries a free_mask (DMatrix<bool>) that records which off-diagonal entries are free parameters versus structural zeros from block_omega declarations.

Internal parameterisation

The outer optimizer works on an unconstrained packed vector x:

x = [log(theta_free), chol(Omega_BSV), log(sigma), chol(Omega_IOV)]
  • Theta: log-transformed to enforce positivity; unpacked with exp()
  • Omega: lower Cholesky factor; diagonal elements log-transformed; unpacked with L * L'
  • Sigma: log-transformed, same as theta

This ensures the optimizer never visits invalid parameter regions (negative variances, non-positive-definite omega). Parameter scaling (scale_params = true) additionally normalizes the packed vector to O(1) before each optimizer call to improve conditioning when parameters span several orders of magnitude.

Parallelism

The outer optimizer evaluates subjects in parallel via rayon:

let subject_nlls: Vec<f64> = population
    .subjects
    .par_iter()
    .map(|subject| foce_subject_nll(subject, &params, &model))
    .collect();

The inner optimizer (EBE search) is per-subject and single-threaded; no nested parallelism is used. Thread count is controlled by FitOptions::n_threads, forwarded to rayon’s global thread pool at the start of each fit call.
