Get started

Quick install

Prerequisites: R ≥ 4.2, cmake, and Rust stable (via rustup).

The standard build uses FERX_NO_AUTODIFF=1, which skips the Enzyme toolchain and falls back to finite-difference gradients. Estimates are identical; compile time is ~2 minutes. Enzyme (exact autodiff) is optional and documented below.

macOS

brew install cmake
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

RStudio on macOS does not inherit the Homebrew PATH. Add cmake explicitly before installing:

# Apple Silicon (M1/M2/M3):
Sys.setenv(PATH = paste("/opt/homebrew/opt/cmake/bin", Sys.getenv("PATH"), sep = ":"))
# Intel Mac — use this line instead:
# Sys.setenv(PATH = paste("/usr/local/opt/cmake/bin", Sys.getenv("PATH"), sep = ":"))

Sys.setenv(FERX_NO_AUTODIFF = "1")
devtools::install_github("FeRx-NLME/ferx-r")

Windows

Install Rtools44 and Rust via rustup. cmake ships with Rtools44.

Sys.setenv(FERX_NO_AUTODIFF = "1")
devtools::install_github("FeRx-NLME/ferx-r")

Linux

sudo apt-get install cmake   # Debian / Ubuntu
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
Sys.setenv(FERX_NO_AUTODIFF = "1")
devtools::install_github("FeRx-NLME/ferx-r")

For full autodiff accuracy, a one-time Enzyme toolchain build is required. Pick your platform:

  • Linux — fully supported, ~30 min plugin build
  • macOS — supported with caveats
  • Windows — not natively supported; use WSL2, Docker, or finite-difference fallback
  • Docker — Enzyme pre-installed, recommended for Windows autodiff
  • Multi-user Linux server

Linux

Tested on Ubuntu 22.04. Other distributions (Debian, Fedora, Arch) work with adjustments to the package manager commands.

1. Install rustup + nightly

Warning

Do not use snap’s rustup — its filesystem confinement breaks on non-standard home directories common on servers.

# Remove snap rustup if present:
sudo snap remove rustup

# Official installer
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y
source "$HOME/.cargo/env"

rustup toolchain install nightly

2. Install system dependencies + matching LLVM

First check which LLVM major version nightly needs:

rustc +nightly --version --verbose | grep LLVM
# e.g. "LLVM version: 22.1.2" — use major = 22

Use that <MAJOR> below:

sudo apt install -y cmake ninja-build clang libssl-dev pkg-config \
                    python3 build-essential curl git libzstd-dev

# Install matching LLVM from apt.llvm.org (Ubuntu's defaults often lag):
wget https://apt.llvm.org/llvm.sh
chmod +x llvm.sh
sudo ./llvm.sh <MAJOR>

sudo chmod 644 /etc/apt/trusted.gpg.d/apt.llvm.org.asc
sudo apt update
sudo apt install -y llvm-<MAJOR>-dev clang-<MAJOR>

For Fedora/RHEL: sudo dnf install llvm<MAJOR>-devel clang<MAJOR>.

3. Build the Enzyme plugin (~15–30 min)

git clone https://github.com/EnzymeAD/Enzyme /tmp/enzyme-build
cd /tmp/enzyme-build/enzyme
mkdir build && cd build

cmake -G Ninja .. \
  -DLLVM_DIR=/usr/lib/llvm-<MAJOR>/lib/cmake/llvm \
  -DENZYME_CLANG=OFF \
  -DENZYME_FLANG=OFF
ninja

4. Install the plugin into nightly’s sysroot

SYSROOT=$(rustc +nightly --print sysroot)
TARGET=x86_64-unknown-linux-gnu   # or aarch64-unknown-linux-gnu on ARM

cp /tmp/enzyme-build/enzyme/build/Enzyme/LLVMEnzyme-<MAJOR>.so \
   $SYSROOT/lib/rustlib/$TARGET/lib/libEnzyme-<MAJOR>.so

Note the filename rewrite: LLVMEnzyme-<N>.so → libEnzyme-<N>.so (the lib prefix is required).

5. Register the toolchain as enzyme

rustup toolchain link enzyme "$(rustc +nightly --print sysroot)"

6. Verify

rustc +enzyme -Z autodiff=Enable - </dev/null 2>&1 | head

Expected output: error[E0601]: 'main' function not found. That is the success signal. If you see autodiff backend not found in the sysroot, the .so is missing or placed in the wrong directory (step 4).

macOS

Supported on both Intel and Apple Silicon.

1. Install Xcode Command Line Tools

xcode-select --install

2. Install rustup + nightly

curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y
source "$HOME/.cargo/env"
rustup toolchain install nightly

3. Install matching LLVM via Homebrew

rustc +nightly --version --verbose | grep LLVM
# note the major version, e.g. 22
brew install llvm@<MAJOR> cmake ninja

Homebrew ships a limited set of LLVM versions. If your version is unavailable, install a slightly older dated nightly that matches what Homebrew has:

rustup toolchain install nightly-YYYY-MM-DD

4. Build the Enzyme plugin

git clone https://github.com/EnzymeAD/Enzyme /tmp/enzyme-build
cd /tmp/enzyme-build/enzyme
mkdir build && cd build

# Adjust prefix: /opt/homebrew (Apple Silicon) or /usr/local (Intel)
cmake -G Ninja .. \
  -DLLVM_DIR=$(brew --prefix)/opt/llvm@<MAJOR>/lib/cmake/llvm \
  -DENZYME_CLANG=OFF \
  -DENZYME_FLANG=OFF
ninja

5. Install the plugin

SYSROOT=$(rustc +nightly --print sysroot)

# Apple Silicon:
TARGET=aarch64-apple-darwin
# Intel: TARGET=x86_64-apple-darwin

# macOS uses .dylib, not .so
cp /tmp/enzyme-build/enzyme/build/Enzyme/LLVMEnzyme-<MAJOR>.dylib \
   $SYSROOT/lib/rustlib/$TARGET/lib/libEnzyme-<MAJOR>.dylib

6. Register and verify

rustup toolchain link enzyme "$(rustc +nightly --print sysroot)"
rustc +enzyme -Z autodiff=Enable - </dev/null 2>&1 | head
# Expect: error[E0601]: `main` function not found
Note

macOS caveats

  • Code-signing errors loading the .dylib: run sudo codesign --force --sign - <path-to-libEnzyme.dylib> — rare on developer machines.
  • Apple Silicon nightly lag: ARM nightlies occasionally trail x86_64 by a day or two; if LLVM versions mismatch after rustup update, pin a specific dated nightly.

Windows

Windows is not currently supported for the full Enzyme autodiff backend. Two practical options are available:

Option A — WSL2 (recommended for full autodiff)

Install Ubuntu under WSL2 and follow the Linux instructions. Performance is near-native for CPU-bound workloads like model fitting.

# From PowerShell (admin):
wsl --install -d Ubuntu-22.04

Then open a WSL terminal and follow the Linux steps above.

Option B — Native Windows (without Enzyme, finite differences only)

Gradients fall back to finite differences — slower and less accurate than autodiff, but the R package is otherwise fully functional.

Prerequisites:

  • R and Rtools44 (ships MinGW gcc compatible with R’s linker)
  • rustup with the GNU-ABI toolchain:
rustup toolchain install stable-x86_64-pc-windows-gnu

You do not need to make it the default — the package build pins this toolchain automatically on Windows.

Then install in R:

devtools::install_github("FeRx-NLME/ferx-r")

Option C — Docker (full autodiff on Windows)

The Docker image bundles the Enzyme toolchain, ferx CLI, the R package, and RStudio Server. See Docker below.

Note

Full native Windows support is blocked on upstream Enzyme integration in rustup (rust-lang/rust#124509). Once that lands, the plugin distribution problem disappears and Windows support becomes straightforward.

Docker

The Docker image ships with the full Enzyme toolchain pre-built — no local Rust or Enzyme setup required. Recommended for Windows users who need autodiff, or any environment where you want a reproducible, isolated setup.

# Build the image (first build ~45–60 min; subsequent builds use the cache)
docker build -t ferx:latest https://github.com/FeRx-NLME/ferx-r.git#main

# Run RStudio Server
docker run --rm -p 8787:8787 -e PASSWORD=ferx ferx:latest

Open http://localhost:8787 — user: rstudio, password: ferx.

To mount a local data directory:

docker run --rm -p 8787:8787 -e PASSWORD=ferx \
  -v /path/to/your/data:/home/rstudio/data \
  ferx:latest

Multi-user Linux server

For shared servers, build the toolchain once and stage it in /opt/rust-nightly so all users share a single installation.

# As root (after completing the Linux steps above):
sudo mkdir -p /opt/rust-nightly
sudo cp -a ~/.rustup/toolchains/nightly-x86_64-unknown-linux-gnu/. /opt/rust-nightly/
sudo chown -R root:root /opt/rust-nightly
sudo chmod -R a+rX /opt/rust-nightly
sudo chmod a+rx /opt/rust-nightly/bin/*

Each user links the shared toolchain into their own rustup:

curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y --default-toolchain none
rustup toolchain link enzyme /opt/rust-nightly

Add to each user’s ~/.Renviron:

PATH=/opt/rust-nightly/bin:${HOME}/.cargo/bin:${PATH}
RUSTUP_TOOLCHAIN=enzyme

See INSTALL-SYSADMIN.md for quarterly refresh procedures and troubleshooting shared installs.

After toolchain setup

Once your Enzyme toolchain is in place, install the R package:

devtools::install_github("FeRx-NLME/ferx-r")

Then run the warfarin example:

library(ferx)
ex  <- ferx_example("warfarin")
fit <- ferx_fit(ex$model, ex$data)
print(fit)
summary(fit)

To see all bundled examples:

ferx_example()   # returns a character vector of all available example names
?ferx_example    # full list with descriptions in the help page

Troubleshooting

"autodiff backend not found in the sysroot" The .so/.dylib is missing or in the wrong location. It must be in <sysroot>/lib/rustlib/<target>/lib/libEnzyme-<N>.so — not just lib/.

"error: the option 'Z' is only accepted on the nightly compiler" Your shell (or R) is finding a stable rustc. Check rustc --version, your PATH, and ~/.Renviron.

"custom toolchain 'enzyme' is not installed" Run: rustup toolchain link enzyme $(rustc +nightly --print sysroot)

"incorrect value 'X' for unstable option 'autodiff'" Valid values change between nightlies. Verify with:

rustc +enzyme -Z autodiff=Enable - </dev/null 2>&1 | head

For more troubleshooting detail see the ferx-core installation docs.
