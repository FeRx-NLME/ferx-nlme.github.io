Get started

Two install paths

ferx has two builds. They produce the same parameter estimates — they differ in how gradients are computed and how fast fits run.

Build Gradients Compile time Fit speed When to use
Simple (default) Finite differences ~2 min Baseline Trying ferx out, Windows-native, no Rust nightly
Full (Enzyme autodiff) Exact via Enzyme ~45–60 min one-time toolchain build Faster, more accurate Hessians Production fits, large models, SIR / uncertainty quantification

You can start with the simple build and upgrade later — the R API is identical.

For a deeper walkthrough of either path, see the installation chapter of the ferx-r book.

What you give up with the simple build

Almost nothing is disabled — the simple build falls back to finite-difference (FD) gradients everywhere autodiff would be used:

Feature Simple build
ferx_simulate(), ferx_predict() Identical — no gradients used
ODE / SDE / EKF models Identical — these always use FD, even in the full build
SAEM Identical — derivative-free E-step
Population (outer-loop) optimizer Identical — always FD in both builds
FOCE / FOCEI inner loop with analytical 1-/2-cpt PK Slower — FD instead of Enzyme reverse-mode AD (≈ 2 × n_eta extra gradient calls per iteration); results match
Hessian-based standard errors Slightly less accurate (FD Hessian)

If you mostly fit ODE models or use SAEM, the full build buys you nothing. The full build’s main win is faster FOCEI on analytical-PK models and crisper SEs for SIR / uncertainty quantification work. Tracked in ferx-r#46.

Pick a version

The install snippets below default to main (development). For reproducible work, pin to the latest tagged release — currently v0.1.5:

# Latest release (recommended for new users / reproducible work)
pak::pak("FeRx-NLME/ferx-r@v0.1.5")

# Latest development version (tracks main; picks up new features and fixes)
pak::pak("FeRx-NLME/ferx-r")

@v0.1.5 works with remotes::install_github() and devtools::install_github() too. See the ferx-r releases page for what each release ships and a link to the matching ferx-core engine release.

Simple build

The simple build skips the Enzyme toolchain entirely. It works on plain stable Rust and falls back to finite-difference gradients. Estimates match the full build; fits are somewhat slower and standard errors from the FD Hessian are slightly less accurate.

Prerequisites: R ≥ 4.2, cmake, and Rust stable (via rustup).

macOS

brew install cmake
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

RStudio on macOS does not inherit the Homebrew PATH. Add cmake explicitly before installing:

# Apple Silicon (M1/M2/M3):
Sys.setenv(PATH = paste("/opt/homebrew/opt/cmake/bin", Sys.getenv("PATH"), sep = ":"))
# Intel Mac:
# Sys.setenv(PATH = paste("/usr/local/opt/cmake/bin", Sys.getenv("PATH"), sep = ":"))

Sys.setenv(FERX_NO_AUTODIFF = "1")
pak::pak("FeRx-NLME/ferx-r")

Windows

Install Rtools44 and Rust via rustup (cmake ships with Rtools44), then in R:

Sys.setenv(FERX_NO_AUTODIFF = "1")
pak::pak("FeRx-NLME/ferx-r")

Linux

sudo apt-get install cmake   # Debian / Ubuntu
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
Sys.setenv(FERX_NO_AUTODIFF = "1")
pak::pak("FeRx-NLME/ferx-r")

The FERX_NO_AUTODIFF=1 flag tells the package build to skip the Enzyme probe and go straight to the stable, finite-difference path.

Full build (Enzyme autodiff)

The full build uses Enzyme to compute exact gradients via automatic differentiation. This requires building a custom rustc from source with Enzyme integrated into its own LLVM — a one-time ~45–60 min setup that needs ~30 GB of free disk.

Important

Do not build a standalone Enzyme plugin against a separately-installed LLVM and copy it into a prebuilt nightly’s sysroot. That setup looks like it works but causes Enzyme to hang silently in llvm::Constant::getNullValue the first time it actually differentiates anything. The full toolchain must be built from a single rust-lang/rust source tree with --enable-llvm-enzyme and --set llvm.download-ci-llvm=false. See the warning at the top of the ferx-core installation page.

1. Build the Enzyme toolchain

Follow the platform steps in the ferx-core book:

  • Linux
  • macOS (Intel and Apple Silicon)
  • Windows — not supported; use WSL2 or Docker

Then verify the toolchain with a real #[autodiff_forward] test — the usual E0601 check is not sufficient and passes for broken toolchains.

For shared servers, see multi-user installs.

2. Install the R package

Once rustc +enzyme is verified working:

pak::pak("FeRx-NLME/ferx-r")

The package’s src/Makevars auto-detects the enzyme toolchain and builds the autodiff path. To confirm afterwards:

ferx:::ferx_rust_autodiff_enabled()
# [1] TRUE

To force one path or the other, set before installing:

  • FERX_NO_AUTODIFF=1 — force the simple build even if Enzyme is installed
  • FERX_NO_AUTODIFF=0 — force the full build, erroring out if Enzyme isn’t found

Docker (full autodiff, no local toolchain)

The ferx-r Docker image ships the Enzyme toolchain prebuilt — the fastest way to get a working full-autodiff setup, especially on Windows.

docker build -t ferx:latest https://github.com/FeRx-NLME/ferx-r.git#main
docker run --rm -p 8787:8787 -e PASSWORD=ferx ferx:latest

Open http://localhost:8787 — user: rstudio, password: ferx.

First fit

After either install, run the bundled warfarin example:

library(ferx)
ex  <- ferx_example("warfarin")
fit <- ferx_fit(ex$model, ex$data)
print(fit)
summary(fit)

List all bundled examples:

ferx_example()   # character vector of available example names
?ferx_example    # full list with descriptions
